###工厂模式
工厂模式将对应的实际类的初始化封装在方法中，让子类来决定初始化哪个类 :

+ 组合复杂的对象来创建代码

+  选择一个类来初始化

+  缓存对象

+  协调对共享资源的访问

Java 中使用工厂模式

	public interface Animal{
		void say();
	}
	
	public class Dog implements Animal{
		private Dog(){}
		public void say(){
			System.out.println("I ' m a dog !");
		}
	}
	public classs Cat implements Animal{
		private Cat(){}
		public  void say(){
			System.out.println("I ' m a cat !");
		}
	}

	public class AnimalFactory{
		public static Animal createAnimal(String kind){
			if("cat".equals(kind)){
				return new Cat();
			}else if("dog".equals(kind)){
				return new Dog();
			}else{
				return null;
			}
		}
	}


在Scala中我们可以这样实现
Scala 代码

	trait Animal{
		def say
	}
	class Dog extends Animal{
		override def say{
			printf("I ' m a dog !");
		}
	}
	class Cat extends Animal{
		override def say{
			printf("I ' m a cat !");
		}
	}

	object Animal{
		//apply命名好处是我们调用时可以省略掉.apply
		def apply( kind : String ) = kind match{
			case "dog" => new Dog()
			case "cat" => new Cat();
		}
	}

	object Tese extends App{
		val animal = Animal("dog");
		animal say //I ' m a dog !
	}

优点:

+ 重用基类名称

+ 标准且简洁

+ 类似于构造器调用

缺点：

+ 仅限于静态工厂方法
